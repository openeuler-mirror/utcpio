`utcpio`（复制进/出）是一个用于创建和提取归档文件的命令行工具。

以下是 `cpio` 的主要用法：

**1. 创建归档 (copy-out 模式)**

   * **语法:** `find <目录> | cpio -o[选项] > <归档文件>`
   * **示例:**
      ```bash
      find ./mydir | cpio -o > myarchive.cpio
      ```
      这将把 `mydir` 目录及其所有子目录和文件打包到 `myarchive.cpio` 文件中。

**2. 列出归档内容 (copy-in 模式)**

   * **语法:** `cpio -itv < <归档文件>`
   * **示例:**
      ```bash
      cpio -itv < myarchive.cpio
      ```
      这将列出 `myarchive.cpio` 文件的内容，包括文件名、权限和大小。

**3. 提取归档内容 (copy-in 模式)**

   * **语法:** `cpio -idmv < <归档文件>`
   * **示例:**
      ```bash
      cpio -idmv < myarchive.cpio
      ```
      这将把 `myarchive.cpio` 文件中的内容提取到当前目录。

**4. 复制文件到另一个目录 (copy-pass 模式)**
   * **语法:** `cpio --pvd  <指定目录>`
   * **示例:**
      ```bash
      find . -print | cpio -pvd /path/to/destination      
      ```
**5. 创建硬链接而不是复制文件 (copy-pass 模式)**
   * **语法:** `cpio --plvd  <指定目录>`
   * **示例:**
      ```bash
      find . -print | cpio -plvd /path/to/destination
      ```

**常用选项：**

* **-o (copy-out):** 创建归档。
* **-i (copy-in):** 提取归档。
* **-t (list):** 列出归档内容。
* **-v (verbose):** 显示详细输出。
* **-d (make directories):** 必要时创建目录。
* **-m (preserve modification time):** 保留文件的修改时间。
* **-c (ASCII header):** 使用 ASCII 头部，提高可移植性。
* **-O <归档文件> (output file):** 指定输出归档文件。
* **-I <归档文件> (input file):** 指定输入归档文件。
* **-F <归档文件> :** 指定要提取的归档文件。



**注意事项：**

* `cpio` 通常与 `find` 命令结合使用，以指定要归档的文件。
* `cpio` 不会自动压缩归档文件，需要使用 `gzip` 或其他压缩工具。
* `cpio` 默认情况下不会覆盖已存在的文件，可以使用 `-u` 选项覆盖。

**优点：**

* `cpio` 可以处理任何类型的文件，包括设备文件和符号链接。
* `cpio` 可以跨越多个磁盘或磁带。
* `cpio` 可以创建增量备份。

2 教程
**********

GNU 'utcpio' 执行三个主要功能。将文件复制到归档文件中，从归档文件中提取文件，
以及将文件传递到另一个目录树。归档文件可以是磁盘上的文件、一个或多个软盘，
或一个或多个磁带。

   创建归档文件时，'utcpio' 从标准输入获取要处理的文件列表，然后将归档文件发送到
标准输出，或发送到由 '-F' 选项定义的设备。*注意 复制输出模式::。
通常使用 'find' 或 'ls' 将此列表提供给标准输入。在以下示例中，
您可以看到归档单个目录内容的可能性。

     % ls | utcpio -ov > directory.utcpio

   '-o' 选项创建归档文件，'-v' 选项在添加文件时打印文件名。
请注意，选项可以放在单个 '-' 后面，也可以单独放在命令行上。
'>' 将 'utcpio' 输出重定向到文件 'directory.utcpio'。

   如果要归档整个目录树，'find' 命令可以向 'utcpio' 提供文件列表：

     % find . -print -depth | utcpio -ov > tree.utcpio

   这将获取当前目录中的所有文件、下面的目录，并将它们放入归档文件 'tree.utcpio' 中。
同样，'-o' 创建归档文件，'-v' 选项在归档时显示文件名。
*注意 复制输出模式::。在 'find' 语句中使用 '.' 在执行恢复时提供更大的灵活性，
因为它会保存带有相对路径的文件名，而不是硬编码的绝对路径。
'-depth' 选项强制 'find' 在打印目录本身之前打印目录中的条目。
这通过在处理目录名称本身之前打印目录中的条目来限制限制性目录权限的影响。

   提取归档文件需要更多的思考。首先，默认情况下 'utcpio' 提取的文件名与归档中存储的
文件名完全相同。这意味着如果归档包含绝对路径，无论您在运行命令时位于哪个目录，
都将提取文件到它们的绝对位置。您可以使用 '--no-absolute-filenames' 选项指示 'utcpio'
删除前导斜杠。尽管如此，良好的做法是在提取之前始终使用 'utcpio -t' 测试归档文件。

   此外，'utcpio' 默认情况下不会创建目录。另一个特点是除非您告诉它，
否则它不会覆盖现有文件。

     % utcpio -iv < directory.utcpio

   这将检索文件 'directory.utcpio' 中归档的文件并将它们恢复到它们的位置。
'-i' 选项提取归档文件，'-v' 在提取时显示文件名。如果您正在处理归档的目录树，
则需要使用 '-d' 选项根据需要创建目录，如下所示：

     % utcpio -idv < tree.utcpio

   这将获取归档文件 'tree.utcpio' 的内容并提取它。如果您尝试提取的文件与已存在的
同名文件（并且具有相同或更新的修改时间）重叠，'utcpio' 将不会提取该文件，
除非通过 '-u' 选项告诉它这样做。*注意 复制输入模式::。

   在复制传递模式下，'utcpio' 将文件从一个目录树复制到另一个目录树，
结合复制输出和复制输入步骤而不实际使用归档文件。它从标准输入读取要复制的文件列表；
将它们复制到的目录作为非选项参数给出。*注意 复制传递模式::。

     % find . -depth -print0 | utcpio --null -pvd new-dir

   该示例显示将当前目录和子目录的文件复制到名为 new-dir 的新目录中。
一些新选项是 GNU 'find' 可用的 '-print0'，与 'utcpio' 的 '--null' 选项结合使用。
这两个选项共同作用，即使在文件名中嵌入了特殊字符，也能在 'find' 和 'utcpio' 之间发送文件名。
另一个是 '-p'，它告诉 'utcpio' 将其找到的文件传递到目录 'new-dir'。

     保留文件修改时间。在创建文件时保留归档中记录的修改时间，而不是使用当前时间。
     此选项在复制输入和复制传递模式下有效。

'-m'
     等同于 '--preserve-modification-time'。

'--no-preserve-modification-time'
     [*note copy-in::,*note copy-pass::]
     不保留文件修改时间（默认行为）。创建文件时使用当前时间作为修改时间。

'-n'
'--numeric-uid-gid'
     [*note copy-in::]
     在详细的目录内容列表中，显示数字 UID 和 GID 值，而不是尝试解析为用户名和组名。

'--only-verify-crc'
     [*note copy-in::]
     当读取 CRC 格式的归档文件时，仅验证归档中每个文件的 CRC 校验值，而不实际提取文件。

'--quiet'
     [*note copy-in::,*note copy-out::,*note copy-pass::]
     不显示复制的块数。

'--rsh-command=COMMAND'
     [*note copy-in::,*note copy-out::,*note copy-pass::]
     使用指定的 COMMAND 代替 'rsh' 来访问远程归档。

'-r'
'--rename'
     [*note copy-in::]
     交互式重命名文件。在提取每个文件前提示输入新文件名。

'--sparse'
     [*note copy-in::]
     将包含大块零的文件写入为稀疏文件，以节省磁盘空间。

'-s'
'--swap-bytes'
     [*note copy-in::]
     交换文件中每个半字（2字节）的字节顺序。

'-S'
'--swap-halfwords'
     [*note copy-in::]
     交换文件中每个字（4字节）的半字顺序。

'--to-stdout'
     [*note copy-in::]
     将提取的文件内容输出到标准输出，而不是写入磁盘文件。

'-u'
'--unconditional'
     [*note copy-in::]
     无条件替换所有文件。覆盖现有文件而不检查修改时间。

'-v'
'--verbose'
     [*note copy-in::,*note copy-out::,*note copy-pass::]
     显示详细处理信息。在处理文件时列出文件名。

'-V'
'--dot'
     [*note copy-in::,*note copy-out::,*note copy-pass::]
     为每个处理的文件打印一个 '.' 作为进度指示。

'-W'
'--warning=FLAG'
     [*note copy-in::,*note copy-out::,*note copy-pass::]
     控制警告显示。参数可以是 'none'、'truncate'、'no-truncate' 或 'all'。
     详情请参阅 *note warning:: 部分。

5 报告错误或建议
*******************************

您可能会遇到'utcpio'中的错误。如果发生这种情况，我们希望能获知相关信息。由于错误报告的目的是改进软件，请在报告时务必包含以下完整信息：

   • 您使用的软件包版本
   • 配置软件包时使用的编译选项
   • 错误出现的具体条件

   请将报告发送至<bug-utcpio@gnu.org>。这是一个公开邮件列表；所有发送至此的邮件都会被存档并可在<https://lists.gnu.org/archive/html/bug-utcpio>查看。您的错误报告也会在那里公开。该列表不仅限于错误报告，实际上它专门用于任何关于GNU 'utcpio'的技术讨论。如需订阅，请访问<https://lists.gnu.org/mailman/listinfo/bug-utcpio>。
